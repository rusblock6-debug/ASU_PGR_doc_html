def answer_with_reasoning(question):
    # Шаг 1: Первый поиск
    chunks_1 = search(question)
    
    # Шаг 2: Phi-4 анализирует и решает нужен ли ещё поиск
    analysis = phi4.analyze(f"""
    Вопрос: {question}
    Найденная информация: {chunks_1}
    
    Достаточно ли информации для ответа?
    Если нет - какие ещё темы нужно найти?
    
    Формат:
    ДОСТАТОЧНО: да/нет
    НУЖНО НАЙТИ: [список тем]
    """)
    
    # Шаг 3: Если нужно - делаем дополнительные поиски
    all_chunks = chunks_1
    if not analysis.sufficient:
        for topic in analysis.topics_needed:
            additional_chunks = search(topic)
            all_chunks.extend(additional_chunks)
    
    # Шаг 4: Финальный ответ с рассуждением
    answer = phi4.generate_with_reasoning(question, all_chunks)
    
    return answer

## 2. ПРОИЗВОДИТЕЛЬНОСТЬ (объясняю проще)

**Сейчас:**
- Пользователь спрашивает "Как построить схему?"
- Поиск в ChromaDB → 20 чанков
- Отправляем в Phi-4 → генерация ответа
- Кэшируем **финальный ответ** в Redis

**Проблема:**
Если другой пользователь спросит "Как создать схему дорог?" (похожий вопрос):
- Поиск снова → те же 20 чанков
- Phi-4 снова генерирует (30 секунд)

**Решение - кэшировать результаты поиска:**
```python
# Кэш поиска
search_cache_key = f"search:{hash(question)}"
cached_chunks = redis_client.get(search_cache_key)

if cached_chunks:
    chunks = json.loads(cached_chunks)
else:
    chunks = search(question)
    redis_client.setex(search_cache_key, 3600, json.dumps(chunks))
```

**Профит:** Поиск мгновенный, только генерация 30 сек
