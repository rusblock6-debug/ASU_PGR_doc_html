# Улучшения поиска в index.html

## Исправленные проблемы

### 1. Исправлено экранирование специальных символов в регулярных выражениях
**Проблема:** В функциях `highlightQuery` и `highlightTextInContent` использовались UUID-строки вместо правильного экранирования `\\$&`.

**Исправление:** Заменено на правильное экранирование:
```javascript
const esc = query.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
```

### 2. Улучшен алгоритм поиска
**Проблема:** Использовался простой `includes()`, который искал только точное совпадение подстроки.

**Улучшения:**
- Добавлена система оценки (scoring) результатов
- Поиск по нескольким словам (разделённым пробелами)
- Приоритизация результатов:
  - Точное совпадение с заголовком: 100 баллов
  - Заголовок начинается с запроса: 50 баллов
  - Совпадение целого слова в заголовке: 40 баллов
  - Частичное совпадение в заголовке: 30 баллов
  - Совпадение целого слова в тексте: 15 баллов
  - Частичное совпадение в тексте: 10 баллов
  - Бонус за совпадение всех слов запроса: +20 баллов

### 3. Добавлен debounce для поиска
**Проблема:** Поиск выполнялся при каждом нажатии клавиши, что могло вызывать задержки.

**Исправление:** 
- Добавлена функция `handleSearchDebounced()` с задержкой 150мс
- Поиск выполняется только после того, как пользователь перестал печатать

### 4. Улучшено отображение результатов
**Добавлено:**
- Счётчик общего количества результатов
- Счётчик результатов по каждой категории (роли)
- Подсветка поискового запроса в сообщении "ничего не найдено"

## Как использовать

Просто откройте `tetepfgr/index.html` в браузере. Поиск теперь работает:
- Быстрее (благодаря debounce)
- Точнее (благодаря системе оценки)
- Умнее (поиск по нескольким словам, приоритизация результатов)

## Примеры поиска

- `вход` - найдёт "Вход в систему", "Авторизация" и т.д.
- `настройка системы` - найдёт все результаты, содержащие оба слова
- `меню` - найдёт "Главное меню", "Меню пользователя" и т.д.
