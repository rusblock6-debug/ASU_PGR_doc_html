### ШАГ 0: SQLite для метаданных и истории (инфраструктура)

**Зачем нужно:**
- ChromaDB хранит чанки + эмбеддинги (векторный поиск)
- SQLite хранит структурированные данные (история, фидбек, метрики)

**Что храним в SQLite:**
1. **История чатов** (когда будем делать UI чата):
   - Сессии пользователей
   - Сообщения (вопросы и ответы)
   - Временные метки

2. **Фидбек пользователей**:
   - 👍/👎 оценки ответов
   - Комментарии к негативным оценкам
   - Какие чанки использовались

3. **Метрики качества**:
   - Время поиска
   - Время генерации ответа
   - Уверенность бота
   - Cache hit rate

4. **Кэш извлечённых знаний** (для иерархической индексации):
   - MD5 файла → описание от Phi-4
   - Чтобы не пересчитывать неизменённые файлы

**Почему SQLite а не PostgreSQL:**
- ✅ Один файл (не нужен отдельный сервер)
- ✅ Встроен в Python (не нужно устанавливать)
- ✅ Достаточно для наших задач
- ✅ Легко мигрировать на PostgreSQL потом если нужно

**Структура БД:**

```sql
-- Таблица сессий

-- Таблица фидбека

    issues JSON,  -- ["irrelevant", "incomplete", ...]
    comment TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Таблица метрик
CREATE TABLE metrics (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    question TEXT,
    search_time REAL,
    generation_time REAL,
    confidence REAL,

-- Таблица кэша знаний (для иерархической индексации)
```

## ЧАСТЬ 2: Гибридный поиск

**Комбинируем 3 типа поиска:**

### 1. Векторный поиск (semantic)
```python
vector_results = collection.query(
    query_texts=[question],
    n_results=10
)
```
Находит по смыслу, понимает контекст.

### 2. Keyword поиск (exact match)
```python
# Извлекаем ключевые слова из вопроса
keywords = extract_keywords(question)  # ["метка", "добавить"]

# Ищем точные совпадения в тексте чанков
keyword_results = collection.query(
    where_document={"$or": [
        {"$contains": "метка"},
        {"$contains": "метки"}
    ]},
    n_results=10
)
```
Находит точные совпадения слов.

### 3. Synonym expansion (расширение синонимами)
```python
# Расширяем запрос синонимами
expanded_query = expand_with_synonyms(question)
# "как добавить метку" → "как добавить метку место точку узел"

synonym_results = collection.query(
    query_texts=[expanded_query],
    n_results=10
)
```
Находит через синонимы.

### 4. Объединение и Reranking
```python
# Объединяем все результаты
combined = vector_results + keyword_results + synonym_results

# Убираем дубликаты
unique = remove_duplicates(combined)  # ~20-25 чанков

# Rerank через Phi-4 (выбираем топ-5 самых релевантных)
final_results = rerank_with_phi4(unique, question)  # топ-5
```

```python
def normalize_query(query):
    # 1. Lowercase
    query = query.lower()
    
    # 2. Транслитерация
    query = transliterate(query)  # bort → борт
    
    # 3. Удаление лишних символов
    query = re.sub(r'[-_]', ' ', query)  # борт-контроль → борт контроль
    
    # 4. Исправление опечаток (fuzzy matching)
    query = fix_typos(query)  # Борт0кортроль → борт контроль
    
    return query
```

```javascript
// При загрузке страницы
let chats = JSON.parse(localStorage.getItem('ai_chats'));
```

### 3. Автоматическая индексация при первом запуске

**Файл:** `tetepfgr/ai-bot/entrypoint.sh`

```bash
#!/bin/bash
set -e

echo "🚀 Запуск AI-ассистента АСУ ПГР"

# Ждём Ollama
echo "🔄 Ожидание запуска Ollama..."
until curl -s http://ollama:11434/api/tags > /dev/null 2>&1; do
  echo "   Ожидание Ollama..."
  sleep 2
done
echo "✅ Ollama запущен"

# Проверяем и скачиваем модель
echo "📥 Проверка модели ${OLLAMA_MODEL:-phi4-mini}..."
if ollama list | grep -q "${OLLAMA_MODEL:-phi4-mini}"; then
  echo "✅ Модель ${OLLAMA_MODEL:-phi4-mini} уже установлена"
else
  echo "📥 Скачивание модели ${OLLAMA_MODEL:-phi4-mini}..."
  ollama pull ${OLLAMA_MODEL:-phi4-mini}
  echo "✅ Модель загружена"
fi

# Запускаем FastAPI в фоне
echo "🚀 Запуск AI-бота..."
python main.py &
API_PID=$!

# Ждём запуска API
echo "⏳ Ожидание запуска API..."
until curl -s http://localhost:8000/health > /dev/null 2>&1; do
  sleep 2
done
echo "✅ API запущен"

# АВТОМАТИЧЕСКАЯ ИНДЕКСАЦИЯ при первом запуске
if [ ! -f "/app/chroma_data/.indexed" ]; then
  echo ""
  echo "📚 ПЕРВЫЙ ЗАПУСК - Индексация документации"
  echo "   Это займёт 15-30 минут..."
  echo ""
  
  curl -X POST "http://localhost:8000/api/index?mode=full" \
       -H "X-API-Key: ${API_KEY:-change-me-in-production}" \
       -H "Content-Type: application/json" \
       --max-time 7200  # 2 часа таймаут
  
  if [ $? -eq 0 ]; then
    touch /app/chroma_data/.indexed
    echo ""
    echo "✅ Индексация завершена успешно!"
    echo "🎉 AI-ассистент готов к работе: http://localhost:${DOCS_PORT:-3000}"
  else
    echo "❌ Ошибка индексации. Проверьте логи."
  fi
else
  echo "✅ База знаний уже проиндексирована"
  echo "🎉 AI-ассистент готов к работе: http://localhost:${DOCS_PORT:-3000}"
fi

# Держим контейнер запущенным
wait $API_PID
```

### 7. REASONING (Рассуждение и логические выводы)

**Проблема:** Бот не может делать выводы
- Вопрос: "Как сделать дорогу между местами?"
- В базе есть: "Места работ", "Схема дорог", "Узлы"
- Прямого ответа НЕТ, но можно ВЫВЕСТИ логически
- Текущий бот отвечает: "Информация не найдена"

**Решение: Chain-of-Thought (цепочка рассуждений)**

---

## ЧАСТЬ 1: Промпт с рассуждением

**Текущий промпт (простой):**
```python
SYSTEM_PROMPT = """
Контекст: {context}
Вопрос: {question}

Ответь ТОЛЬКО по контексту.
"""
```

**Новый промпт (с рассуждением):**
```python
SYSTEM_PROMPT = """Ты — технический ассистент для системы "АСУ ПГР".

ТВОЯ РОЛЬ:
- Отвечай на вопросы по документации и коду
- Используй информацию из контекста
- ДЕЛАЙ ЛОГИЧЕСКИЕ ВЫВОДЫ если прямого ответа нет
- Связывай информацию из разных частей системы

ПРАВИЛА РАССУЖДЕНИЙ:
1. Если прямой ответ есть в контексте - используй его (уверенность 90-100%)
2. Если прямого ответа нет, но можно ВЫВЕСТИ логически - выведи (уверенность 60-80%)
3. Объясни свою логику в разделе РАССУЖДЕНИЕ
4. Укажи уверенность (ниже если вывод, выше если прямой ответ)
5. Если вывести невозможно - честно скажи "Информация не найдена"

ПРИМЕРЫ ЛОГИЧЕСКИХ ВЫВОДОВ:
- "Места работ" + "Узлы схемы" → места являются узлами на схеме
- "Борт" + "Наряд-задание" → борт получает задание для выполнения
- "Схема дорог" + "Маршрут" → маршрут строится по схеме дорог
- "Погрузка" + "Разгрузка" + "Рейс" → рейс это перемещение от погрузки к разгрузке

ФОРМАТ ОТВЕТА:

РАССУЖДЕНИЕ:
[Пошаговая логика:
1. Что нашёл в контексте
2. Как связал информацию
3. Какой вывод сделал
4. Почему этот вывод логичен]

ОТВЕТ:
[Итоговый ответ для пользователя - понятный, пошаговый]

УВЕРЕННОСТЬ: [0-100%]
ИСТОЧНИКИ: [какие чанки использовал]

---

Контекст из документации:
{context}

Вопрос пользователя: {question}

Ответ:
"""
```

**Вопрос:** "Как сделать дорогу между местами?"

**Контекст:**
- Чанк 1: "Места работ: создание мест погрузки и разгрузки на карте"
- Чанк 2: "Схема дорог: построение дорожной сети из узлов"
- Чанк 3: "Узлы схемы: точки на карте, соединяются дорогами"

**Phi-4 отвечает:**
```

1. В контексте есть "места работ" - это точки на карте (погрузка, разгрузка)
2. Есть "схема дорог" - построение сети из узлов
3. Есть "узлы схемы" - точки которые соединяются дорогами
4. Логический вывод: места работ = узлы на схеме дорог
5. Чтобы сделать дорогу между местами:
   - Места должны быть добавлены как узлы
   - Используется редактор схемы дорог
   - Узлы соединяются дорогами

ОТВЕТ:
Чтобы сделать дорогу между местами:
1. Откройте редактор схемы дорог (раздел Администрирование → Схема дорог)
2. Убедитесь что места работы добавлены как узлы на схеме
3. Выберите инструмент "Соединить узлы" или "Добавить дорогу"
4. Кликните на первое место, затем на второе место
5. Дорога будет создана автоматически между этими узлами

УВЕРЕННОСТЬ: 75%
ИСТОЧНИКИ: Места работ (work_places), Схема дорог (road_schema), Узлы (nodes)

Примечание: Прямой инструкции "как сделать дорогу между местами" нет в документации, 
но вывод логичен на основе описания схемы дорог и мест работ.
```

## ЧАСТЬ 2: Multi-hop reasoning (многошаговое рассуждение)

**Проблема:** Иногда одного поиска недостаточно

**Пример:**
- Вопрос: "Как отправить борт в рейс с грузом?"
- Первый поиск находит: "Рейсы бортов"
- Но не находит: "Наряд-задания", "Виды груза"
- Нужен второй поиск по связанным темам

**Решение: Итеративный поиск**

```python
async def answer_with_multi_hop_reasoning(question: str):
    """Ответ с многошаговым рассуждением"""
    
    # Шаг 1: Первый поиск
    chunks_1 = search(question, n_results=10)
    
    # Шаг 2: Phi-4 анализирует достаточно ли информации
    analysis_prompt = f"""
    Вопрос: {question}
    
    Найденная информация:
    {format_chunks(chunks_1)}
    
    Проанализируй:
    1. Достаточно ли этой информации для полного ответа?
    2. Какие связанные темы нужно найти дополнительно?
    
    Формат ответа:
    ДОСТАТОЧНО: да/нет
    НУЖНО НАЙТИ: [список тем через запятую, или "ничего"]
    ПРИЧИНА: [почему нужны дополнительные темы]
    """
    
    analysis = await phi4_generate(analysis_prompt)
    
    # Шаг 3: Если нужно - делаем дополнительные поиски
    all_chunks = chunks_1
    if not analysis.sufficient:
        for topic in analysis.topics_needed:
            logger.info(f"Дополнительный поиск: {topic}")
            additional_chunks = search(topic, n_results=5)
            all_chunks.extend(additional_chunks)
    
    # Шаг 4: Убираем дубликаты
    unique_chunks = remove_duplicates(all_chunks)
    
    # Шаг 5: Финальный ответ с рассуждением
    answer = await generate_answer_with_reasoning(question, unique_chunks)
    
    return answer
```

**Пример работы:**

**Вопрос:** "Как отправить борт в рейс с грузом?"

**Итерация 1:**
- Поиск: "борт рейс груз"
- Найдено: "Рейсы бортов: создание и управление"
- Phi-4 анализ: "Недостаточно. Нужно найти: наряд-задания, виды груза"

**Итерация 2:**
- Поиск: "наряд-задания"
- Найдено: "Наряд-задания: создание заданий для техники"

**Итерация 3:**
- Поиск: "виды груза"
- Найдено: "Справочник видов груза: параметры и категории"

**Финальный ответ:**
```
РАССУЖДЕНИЕ:
1. Для отправки борта в рейс нужно наряд-задание (найдено в контексте)
2. В наряд-задании указывается вид груза (найдено в контексте)
3. Рейс создаётся автоматически при выполнении задания (найдено в контексте)
4. Логический вывод: создать наряд-задание с грузом → борт получит задание → начнёт рейс

ОТВЕТ:
Чтобы отправить борт в рейс с грузом:
1. Создайте наряд-задание (Оперативная работа → Наряд-задания → Создать)
2. Выберите борт (технику)
3. Укажите вид груза из справочника
4. Укажите место погрузки и разгрузки
5. Сохраните задание
6. Борт получит задание и автоматически начнёт рейс

УВЕРЕННОСТЬ: 85%
```

```python
def validate_indexing():
    results = []
    
    for test in load_ground_truth():
        # Ищем чанки
        search_results = search(test["question"])
        
        # Проверка 1: Нужные чанки найдены
        found_chunks = [r.id for r in search_results]
        chunks_ok = all(chunk in found_chunks for chunk in test["expected_chunks"])
        
        # Проверка 2: Есть нужные ключевые слова
        content = " ".join([r.content for r in search_results])
        keywords_ok = all(kw in content.lower() for kw in test["expected_keywords"])
        
        # Проверка 3: Нет лишнего
        no_noise = all(word not in content.lower() for word in test["must_not_contain"])
        
        # Проверка 4: Генерируем ответ и проверяем уверенность
        answer = generate_answer(test["question"], search_results)
        confidence_ok = answer.confidence >= test["min_confidence"]
        
        results.append({
            "test_id": test["id"],
            "chunks_ok": chunks_ok,
            "keywords_ok": keywords_ok,
            "no_noise": no_noise,
            "confidence_ok": confidence_ok,
            "passed": all([chunks_ok, keywords_ok, no_noise, confidence_ok])
        })
    
    # Отчёт
    passed = sum(1 for r in results if r["passed"])
    total = len(results)
    print(f"✅ Тесты пройдены: {passed}/{total} ({passed/total*100:.1f}%)")
    
    return results
```

## ЧАСТЬ 3: Confidence Score (уверенность в ответе)

**Phi-4 оценивает свою уверенность:**

```python
answer_prompt = f"""
Контекст из документации:
{context}

Вопрос пользователя: {question}

Ответь и укажи УВЕРЕННОСТЬ (0-100%):
- 90-100%: Точно знаю ответ из контекста, все детали есть
- 70-89%: Вероятно правильно, основная информация есть
- 50-69%: Не уверен, информации мало
- 0-49%: Информации недостаточно для ответа

Формат ответа:
ОТВЕТ: [подробный ответ]
УВЕРЕННОСТЬ: [число]%
ИСПОЛЬЗОВАННЫЕ ИСТОЧНИКИ: [список файлов и чанков]
```

```python
class RAGMetrics:
    def __init__(self):
        self.metrics = {
            "search_quality": {
                "avg_relevance_score": [],  # средний score найденных чанков
                "top1_hit_rate": [],  # как часто нужный чанк в топ-1
                "top5_hit_rate": [],  # как часто нужный чанк в топ-5
                "avg_search_time": []  # время поиска
            },
            "answer_quality": {
                "avg_confidence": [],  # средняя уверенность
                "positive_feedback_rate": [],  # % положительных оценок
                "negative_feedback_rate": [],
                "avg_response_time": []  # секунды
            },
            "coverage": {
                "indexed_components": 0,
                "total_chunks": 0,
                "last_update": None,
                "index_size_mb": 0
            },
            "usage": {
                "total_questions": 0,
                "unique_users": 0,
                "cache_hit_rate": [],  # % кэшированных ответов
                "questions_per_day": []
            }
        }
    
    def record_search(self, relevance_score, top1_hit, top5_hit, search_time):
        self.metrics["search_quality"]["avg_relevance_score"].append(relevance_score)
        self.metrics["search_quality"]["top1_hit_rate"].append(1 if top1_hit else 0)
        self.metrics["search_quality"]["top5_hit_rate"].append(1 if top5_hit else 0)
        self.metrics["search_quality"]["avg_search_time"].append(search_time)
    
    def record_answer(self, confidence, response_time, cached):
        self.metrics["answer_quality"]["avg_confidence"].append(confidence)
        self.metrics["answer_quality"]["avg_response_time"].append(response_time)
        if cached:
            self.metrics["usage"]["cache_hit_rate"].append(1)
    
    def get_summary(self):
        return {
            "search_quality": {
                "avg_relevance": np.mean(self.metrics["search_quality"]["avg_relevance_score"]),
                "top1_hit_rate": np.mean(self.metrics["search_quality"]["top1_hit_rate"]),
                "top5_hit_rate": np.mean(self.metrics["search_quality"]["top5_hit_rate"])
            },
            "answer_quality": {
                "avg_confidence": np.mean(self.metrics["answer_quality"]["avg_confidence"]),
                "avg_response_time": np.mean(self.metrics["answer_quality"]["avg_response_time"])
            },
            "usage": {
                "total_questions": self.metrics["usage"]["total_questions"],
                "cache_hit_rate": np.mean(self.metrics["usage"]["cache_hit_rate"])
            }
        }
```

```python
if cached_results:
    results = json.loads(cached_results)

    redis_client.setex(search_cache_key, 3600, json.dumps(results))
```
