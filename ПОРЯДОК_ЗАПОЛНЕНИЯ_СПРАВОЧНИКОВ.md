# Порядок заполнения справочников АСУ ПГР

## Обзор зависимостей

Справочники системы имеют иерархическую структуру с зависимостями. Некоторые справочники ссылаются на другие через внешние ключи (foreign keys).

---

## Правильный порядок заполнения

### 1️⃣ ПЕРВЫЙ УРОВЕНЬ (независимые справочники)

Эти справочники не зависят от других и должны быть заполнены первыми:

#### **Enterprise Service:**
- **Предприятие** (`enterprise_settings`)
  - Базовая информация о предприятии/карьере/шахте
  - Обязательно для всех остальных справочников

- **Модели техники** (`vehicle_models`)
  - Справочник моделей транспортных средств
  - Независимый справочник

- **Типы грузов** (`load_types` / `cargo_types`)
  - Виды перевозимых грузов (руда, порода, уголь и т.д.)
  - Независимый справочник

- **Категории типов грузов** (`load_type_categories`)
  - Группировка типов грузов
  - Независимый справочник

#### **Graph Service:**
- **Шахты/Стволы** (`shafts`)
  - Вертикальные выработки (стволы шахт)
  - Независимый справочник
  - ⚠️ **ВАЖНО: Создать ДО горизонтов!**

---

### 2️⃣ ВТОРОЙ УРОВЕНЬ (зависят от первого уровня)

#### **Graph Service:**
- **Горизонты** (`horizons`)
  - Уровни шахты/карьера на разных высотах
  - **Зависит от:** `shafts` (опционально, связь M2M через `shaft_horizons`)
  - Можно создать без привязки к шахтам, но рекомендуется указать

#### **Enterprise Service:**
- **Техника** (`vehicles`)
  - Транспортные средства (самосвалы, погрузчики)
  - **Зависит от:** 
    - `enterprise_settings` (обязательно)
    - `vehicle_models` (опционально)

- **Режимы работы** (`work_regimes`)
  - Графики работы смен
  - **Зависит от:** `enterprise_settings`

- **Статусы** (`statuses`)
  - Статусы заданий и рейсов
  - **Зависит от:** `enterprise_settings`

---

### 3️⃣ ТРЕТИЙ УРОВЕНЬ (зависят от второго уровня)

#### **Graph Service:**
- **Узлы графа** (`graph_nodes`)
  - Точки на дорожной сети
  - **Зависит от:** `horizons` (обязательно - `horizon_id`)

- **Места** (`places`)
  - Места погрузки/разгрузки/стоянки
  - **Зависит от:**
    - `graph_nodes` (опционально - `node_id`)
    - `cargo_types` из Enterprise Service (опционально - `cargo_type`)
  - ⚠️ Можно создать без узла, но рекомендуется привязать к узлу

---

### 4️⃣ ЧЕТВЕРТЫЙ УРОВЕНЬ (зависят от третьего уровня)

#### **Graph Service:**
- **Рёбра графа** (`graph_edges`)
  - Дороги между узлами
  - **Зависит от:** `graph_nodes` (обязательно - `from_node_id`, `to_node_id`)

- **Метки/Теги** (`tags`) - LEGACY
  - Устаревший способ обозначения мест
  - **Зависит от:** `places` (опционально - `place_id`)

- **Лестницы** (`ladder_nodes`)
  - Вертикальные связи между горизонтами
  - **Зависит от:** `graph_nodes` (обязательно)

---

### 5️⃣ ПЯТЫЙ УРОВЕНЬ (операционные данные)

#### **Enterprise Service:**
- **Сменные задания** (`shift_tasks`)
  - Задания на смену для техники
  - **Зависит от:**
    - `work_regimes`
    - `vehicles`
    - `statuses`

- **Маршрутные задания** (`route_tasks`)
  - Конкретные маршруты в рамках сменного задания
  - **Зависит от:**
    - `shift_tasks`
    - `places` из Graph Service (`place_a_id`, `place_b_id`)
    - `statuses`

---

## Практические рекомендации

### ✅ Минимальный набор для начала работы:

1. **Создать предприятие** (enterprise_settings)
2. **Создать шахты** (shafts) - хотя бы одну
3. **Создать горизонты** (horizons) - привязать к шахтам
4. **Создать узлы** (graph_nodes) на горизонтах
5. **Создать рёбра** (graph_edges) между узлами
6. **Создать места** (places) - привязать к узлам
7. **Создать технику** (vehicles)
8. **Создать режим работы** (work_regimes)
9. **Создать статусы** (statuses)
10. **Создать сменные задания** (shift_tasks)
11. **Создать маршрутные задания** (route_tasks)

### ⚠️ Частые ошибки:

- ❌ Попытка создать горизонт без шахт (можно, но не рекомендуется)
- ❌ Попытка создать узел без горизонта (ошибка FK)
- ❌ Попытка создать ребро без узлов (ошибка FK)
- ❌ Попытка создать route_task без places (ошибка FK)

### 💡 Опциональные связи:

Некоторые связи опциональны (nullable):
- `places.node_id` - можно создать место без узла
- `places.cargo_type` - можно создать место без типа груза
- `vehicles.model_id` - можно создать технику без модели
- `horizons.shafts` - можно создать горизонт без привязки к шахтам

## Проверка заполнения

### Через API:

```bash
# Проверить шахты
GET /api/shafts

# Проверить горизонты
GET /api/horizons

# Проверить узлы на горизонте
GET /api/horizons/{horizon_id}/graph

# Проверить места
GET /api/places

# Проверить технику
GET /api/vehicles
```

### Через UI:

1. **Graph Service Frontend** → Справочники → Горизонты
2. **Graph Service Frontend** → 2D редактор → Выбрать горизонт
3. **Enterprise Service** → Техника
4. **Enterprise Service** → Задания

---

## Импорт данных

Для массового заполнения можно использовать:

- **Graph Service:** `POST /api/import/graph` - импорт графа из JSON
- **Enterprise Service:** Bulk операции для техники и заданий

Пример импорта графа:
```json
{
  "horizon_id": 1,
  "nodes": [...],
  "edges": [...],
  "places": [...]
}
```

---

## Дополнительная информация

- Все справочники имеют пагинацию (параметры `page`, `size`)
- Права доступа контролируются через `auth_lib` (Permission.MAP, Permission.HORIZONS и т.д.)
- Данные общие для всех пользователей (нет персонализации)
- Изменения синхронизируются через SSE (Server-Sent Events)
