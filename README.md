# 🤖 АСУ ПГР — RAG Ассистент

> Умный помощник на базе локальной нейросети, интегрированный в документацию проекта

## ⚡ Быстрый старт

### 1. Запуск веб-сервера документации

```bash
cd tetepfgr
node server.js
```

Откройте: http://localhost:3000/admin.html

### 2. Запуск AI-бота (опционально)

```bash
# В корне проекта
cp .env.example .env
nano .env  # Измените API_KEY

# Запуск через Docker
docker-compose -f docker-compose.rag.yml up -d

# Дождитесь загрузки модели (~10 минут)
docker logs -f pgr-ollama

# Запустите индексацию (~20 минут)
curl -X POST "http://localhost:8000/api/index?mode=full" \
  -H "X-API-Key: ваш-ключ-из-env"
```

## 🎯 Как использовать AI-поиск

1. Откройте документацию: http://localhost:3000/admin.html
2. В левом сайдбаре найдите поле поиска
3. Нажмите кнопку **AI** справа от поля поиска
4. Кнопка станет оранжевой — AI-режим активен
5. Задайте вопрос, например:
   - "Как работает аутентификация?"
   - "Где находится код для GPS?"
   - "Какие микросервисы есть в проекте?"

### Переключение режимов

- **AI активен** (кнопка оранжевая) — вопросы обрабатывает нейросеть
- **AI неактивен** (кнопка серая) — обычный текстовый поиск

## 🐳 Состав Docker

| Сервис | Порт | Описание |
|--------|------|----------|
| **ollama** | 11434 | Локальная нейросеть (qwen2.5:3b) |
| **ai-bot** | 8000 | RAG-движок (поиск + генерация) |
| **redis** | 6379 | Кэш ответов |

## 🔄 Обновление базы знаний

После изменений в коде:

```bash
curl -X POST "http://localhost:8000/api/reindex?mode=incremental" \
  -H "X-API-Key: ваш-ключ"
```

## 🖥️ Требования

**Для документации:**
- Node.js 18+

**Для AI-бота:**
- Docker 20.10+
- Docker Compose 2.0+
- 8 GB RAM (рекомендовано 12 GB)
- 4 CPU ядра
- 20 GB свободного места
