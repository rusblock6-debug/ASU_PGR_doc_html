# Auth Service

Сервис аутентификации и авторизации на базе FastAPI с поддержкой JWT токенов, ролевой моделью доступа и системой разрешений.

## Функциональность

### 🔐 Аутентификация
- **Регистрация пользователей** (`/signup`)
- **Вход в систему** (`/login`) 
- **Обновление токенов** (`/refresh`)
- **Выход из системы** (`/logout`)
- **Верификация токена** (`/verify`)

### 👥 Управление пользователями
- CRUD операции над пользователями
- Назначение ролей пользователям
- Активация/деактивация пользователей
- Получение информации о текущем пользователе (`/me`)

### 🎯 Управление ролями
- Создание, просмотр, обновление и удаление ролей
- Назначение разрешений ролям
- Просмотр пользователей с определенной ролью

### 🔑 Управление разрешениями
- Создание и управление разрешениями
- Проверка прав доступа пользователя
- Получение списка всех разрешений

## API Endpoints

### Аутентификация

| Метод | Endpoint | Описание |
|-------|----------|-----------|
| POST | `/signup` | Регистрация нового пользователя |
| POST | `/login` | Вход в систему |
| POST | `/refresh` | Обновление JWT токенов |
| POST | `/logout` | Выход из системы |
| POST | `/verify` | Проверка валидности токена |
| GET | `/permissions/my` | Получение разрешений текущего пользователя |

### Пользователи (требуются права администратора)

| Метод | Endpoint | Описание |
|-------|----------|-----------|
| POST | `/users` | Создание пользователя |
| GET | `/users` | Получение списка всех пользователей |
| GET | `/users/{user_id}` | Получение пользователя по ID |
| GET | `/me` | Получение текущего пользователя с разрешениями |
| PUT | `/users/{user_id}` | Обновление пользователя |
| DELETE | `/users/{user_id}` | Удаление пользователя |
| PUT | `/users/{user_id}/activate` | Активация пользователя |
| POST | `/users/{user_id}/roles/{role_id}` | Назначение роли пользователю |

### Роли (требуются права администратора)

| Метод | Endpoint | Описание |
|-------|----------|-----------|
| POST | `/roles` | Создание роли |
| GET | `/roles` | Получение всех ролей |
| GET | `/roles/{role_id}` | Получение роли по ID |
| PUT | `/roles/{role_id}` | Обновление роли |
| DELETE | `/roles/{role_id}` | Удаление роли |
| POST | `/roles/{role_id}/permissions/{permission_id}` | Добавление разрешения к роли |
| GET | `/roles/{role_id}/users` | Получение пользователей с определенной ролью |

### Разрешения (требуются права администратора, кроме проверки)

| Метод | Endpoint | Описание |
|-------|----------|-----------|
| POST | `/permissions` | Создание разрешения |
| GET | `/permissions` | Получение всех разрешений |
| GET | `/permissions/{permission_id}` | Получение разрешения по ID |
| POST | `/permissions/check` | Проверка наличия разрешения у пользователя |

## Модели данных

### Пользователь (User)
- `id` - уникальный идентификатор
- `username` - имя пользователя
- `password` - хеш пароля
- `is_active` - статус активности
- `role_id` - ссылка на роль

### Роль (Role)
- `id` - уникальный идентификатор
- `name` - название роли
- `description` - описание роли
- `permissions` - список разрешений

### Разрешение (Permission)
- `id` - уникальный идентификатор
- `name` - название разрешения
- `description` - описание разрешения
