# Руководство по использованию импорта графов в UI

## Где находится кнопка импорта

1. Откройте graph-service frontend
2. На левой панели инструментов найдите кнопку **"📥 Импорт"**
3. Кнопка находится рядом с кнопкой "+ Новый уровень"

## Как импортировать граф

### Вариант 1: Из URL

1. Нажмите кнопку **"📥 Импорт"**
2. Выберите вкладку **"Из URL"**
3. Введите URL внешнего API, например:
   ```
   https://api.qsimmine12-staging.dmi-msk.ru/api/road-net/1
   ```
4. Нажмите **"Предпросмотр"**
5. Проверьте информацию о графе (количество узлов, рёбер, меток)
6. Нажмите **"Импортировать"**

### Вариант 2: JSON данные напрямую

1. Нажмите кнопку **"📥 Импорт"**
2. Выберите вкладку **"JSON данные"**
3. Вставьте JSON с данными графа или нажмите **"Вставить пример JSON"**
4. Нажмите **"Предпросмотр"**
5. Проверьте информацию о графе
6. Нажмите **"Импортировать"**

## Формат данных

### Простой формат (рекомендуется для начала)

```json
{
  "nodes": [
    { "id": 1, "x": 0, "y": 0, "z": -50, "type": "road" },
    { "id": 2, "x": 100, "y": 0, "z": -50, "type": "road" }
  ],
  "edges": [
    { "from": 1, "to": 2 }
  ],
  "tags": [
    {
      "id": 1,
      "x": 0,
      "y": 0,
      "z": -50,
      "radius": 25,
      "name": "Точка A",
      "type": "checkpoint",
      "point_id": "point_a"
    }
  ]
}
```

### Структурированный формат с уровнями

```json
{
  "levels": [
    {
      "name": "Уровень -50м",
      "height": -50,
      "nodes": [...],
      "edges": [...],
      "tags": [...]
    }
  ]
}
```

## Опции импорта

### Перезапись существующих уровней

Если хотите заменить существующие данные на уровне:
- Поставьте галочку **"Перезаписать существующие уровни"**
- Граф будет импортирован с заменой данных

⚠️ **Внимание**: Это удалит все существующие узлы, рёбра и метки на уровнях с такой же высотой!

## Что происходит после импорта

1. **Успешный импорт**:
   - Отображается статистика: сколько создано уровней, узлов, рёбер, меток
   - Автоматически выбирается первый импортированный уровень
   - Граф отображается на canvas
   - Диалог закрывается через 2 секунды

2. **Ошибка импорта**:
   - Отображается сообщение об ошибке
   - Показываются детали ошибок (если есть)
   - Можно попробовать снова или закрыть диалог

## Поддерживаемые форматы

Для просмотра всех поддерживаемых форматов:
1. Откройте диалог импорта
2. Нажмите **"Показать поддерживаемые форматы"**
3. Система покажет:
   - Structured Levels Format
   - Flat Graph Format
   - GeoJSON Format

## Примеры использования

### Пример 1: Импорт простого графа

```json
{
  "nodes": [
    { "id": 1, "x": 0, "y": 0, "z": -50, "type": "road" },
    { "id": 2, "x": 100, "y": 0, "z": -50, "type": "road" },
    { "id": 3, "x": 100, "y": 100, "z": -50, "type": "junction" },
    { "id": 4, "x": 0, "y": 100, "z": -50, "type": "road" }
  ],
  "edges": [
    { "from": 1, "to": 2 },
    { "from": 2, "to": 3 },
    { "from": 3, "to": 4 },
    { "from": 4, "to": 1 }
  ]
}
```

Это создаст квадратную дорогу на уровне -50м.

## Troubleshooting

### Проблема: "Некорректный JSON формат"

**Решение**: 
- Проверьте синтаксис JSON (все скобки закрыты, запятые на месте)
- Используйте JSON валидатор (например, jsonlint.com)
- Попробуйте кнопку "Вставить пример JSON"

### Проблема: "Ошибка при импорте"

**Решение**:
- Проверьте доступность backend (http://localhost:5000)
- Убедитесь что graph-service запущен
- Посмотрите детали ошибки в диалоге

### Проблема: "Уровень уже существует"

**Решение**:
- Поставьте галочку "Перезаписать существующие уровни"
- Или измените высоту (z) узлов в JSON

### Проблема: "Node X not found"

**Решение**:
- Убедитесь что все узлы определены перед рёбрами
- Проверьте что `from` и `to` в edges соответствуют `id` в nodes

## Координаты

⚠️ **Важно**: Graph service использует **Canvas координаты** (метры), НЕ GPS!

- `x, y, z` - могут быть любыми числами (положительными или отрицательными)
- Например: `x: 0, y: 0, z: -50` - корректные координаты
- Для работы с GPS используйте калибровку в настройках
