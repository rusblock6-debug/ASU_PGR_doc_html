⚙️ 3. Принцип работы — по шагам (без домыслов)

📚 Этап А. Индексация — как бот «читает» проект

Важно: Бот сканирует ВЕСЬ репозиторий целиком (всё, что вы ему смонтировали).
Но из сканирования исключается мусор: .git, node_modules, __pycache__, картинки, логи.
🔍 1. Сканирование файлов
ВСЕ файлы (кроме мусора).
✂️ 2. Чанкование
Код → режем на функции/классы.
JSON → режем по объектам.
Документы → по параграфам.
🧮 3. Embedding
Превращаем каждый кусочек в вектор (набор чисел).
💾 4. Сохранение
Вектор + текст + метаданные → ChromaDB.

1 Вы или CI/CD запускаете команду: curl -X POST /api/index?mode=full
2 Бот проходит по всем папкам (включая components/, infrastructure/, tetepfgr/ — всё, что есть).
3 Исключает .git, node_modules, __pycache__ и другие технические папки.
4 Для каждого файла кода — разбивает на функции/классы (с помощью Tree-sitter).
5 Сохраняет в ChromaDB (векторная база) — это и есть «база знаний».
6 После этого бот готов отвечать на вопросы.

▼

💬 Этап Б. Как бот отвечает на вопрос

❓ Вопрос
Пользователь пишет в чат.
🔎 Поиск
Бот ищет в ChromaDB топ-20 похожих кусочков.
🎯 Reranking
Уточняет и выбирает топ-4 самых релевантных.
🧠 Генерация
Нейросеть (Ollama) пишет ответ по этим 4 кусочкам.

1 Пользователь пишет вопрос.
2 Бот проверяет Redis — если такой же вопрос уже был, отвечает мгновенно.
3 Если нет — превращает вопрос в вектор (embedding).
4 Ищет в ChromaDB (базе знаний) самые похожие куски кода/документации (top-20).
5 Reranker (модель bge-reranker) выбирает из них 4 лучших.
6 Отправляет эти 4 кусочка в нейросеть (qwen2.5:3b) с чёткой инструкцией: «отвечай ТОЛЬКО по этим данным».
7 Нейросеть генерирует ответ → пользователь видит его + ссылки на файлы и строки.

⏱️ Время ответа: 12–18 секунд (на CPU).
⚡ Повторные вопросы: мгновенно (кэш Redis).

📸 4. Цифровой снимок + Обновления (CI/CD ready)

Проблема: как понять, что именно проиндексировалось?
Решение: сначала бот делает снимок проекта (snapshot) — JSON-файл со списком ВСЕХ файлов, их хешей и предупреждениями.

Что такое снимок?

{
  "timestamp": "2026-04-27T14:30:00Z",
  "files": [
    {"path": "components/analytics/__main__.py", "hash": "md5...", "size": 3200},
    {"path": "infrastructure/postgres/init.sql", "hash": "md5...", "size": 5400}
  ],
  "excluded": [".git", "node_modules", "__pycache__"]
}

Как вы управляете обновлениями

1️⃣ Создать снимок
curl -X POST /api/snapshot/create
Бот сканирует всё (кроме мусора) → выдаёт JSON.
2️⃣ Проверить снимок
curl /api/snapshot/latest
Вы видите, какие файлы попали в индекс, а какие исключены.
3️⃣ Запустить индексацию
curl -X POST "/api/index?snapshot=latest"
Только после вашей команды бот строит базу знаний.

Автоматика (CI/CD) для нетерпеливых

📦 Push в main
Автоматически создаётся снимок, но индексация НЕ запускается.
🏷️ Создание тега (релиз)
Автоматически полная переиндексация.
⏰ Ежедневно в 3 часа ночи
Инкрементальное обновление (только изменённые файлы).
🔁 Что происходит, когда вы меняете папки или файлы?
— Переименовали папку → сработает reindex → старые векторы удалятся, новые создадутся.
— Добавили новый файл → при следующем reindex он будет проиндексирован.
— Удалили файл → его векторы исчезнут.
Никакой магии — всё детерминировано.
