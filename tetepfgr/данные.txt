Инструкция пользователя АСУ ПГР
1. Введение в систему
Общее описание системы автоматизированного управления подвижным составом горно-рудного предприятия

1.1. О системе АСУ ПГР
Назначение системы
АСУ ПГР (Автоматизированная Система Управления Подвижным Составом Горно-Рудного Предприятия) предназначена для:

Автоматизации диспетчерского управления техникой на руднике

Контроля выполнения плановых показателей по перевозке грузов

Мониторинга работы оборудования в реальном времени

Анализа эффективности использования техники

Оптимизации маршрутов движения и распределения заданий

Система обеспечивает централизованное управление всеми производственными процессами карьера.

https://screenshots/intro/system_overview.png

Основные возможности
Управление справочниками: техника, статусы, виды груза, места работ, горизонты

Планирование сменных заданий через наряд-задания

Отслеживание рейсов техники в реальном времени

Визуализация работы на 2D и 3D картах рудника

Анализ производительности через карту рабочего времени (КРВ)

Формирование отчетов по выполненным работам

Контроль остатков груза на местах погрузки и разгрузки

https://screenshots/intro/features.png

Архитектура системы
Система состоит из следующих основных компонентов:

Бортовое оборудование (БОРТ) — устанавливается на технику для сбора телеметрии

Сервер диспетчеризации — обрабатывает данные и управляет заданиями

Веб-интерфейс диспетчера — основной интерфейс для работы

База данных — хранение всей информации о работе предприятия

Все компоненты работают совместно для обеспечения непрерывного контроля производства.

https://screenshots/intro/architecture.png

1.2. Термины и определения
Основные понятия
ШАС (Шахтный Автосамосвал) — самосвал для подземных работ

ПДМ (Погрузочно-Доставочная Машина) — техника для погрузки и доставки породы

Наряд-задание — план работы техники на смену с набором маршрутных заданий

Маршрутное задание — конкретный маршрут от места погрузки до разгрузки

Рейс — фактическое выполнение маршрутного задания техникой

Горизонт — уровень рудника с определенной высотной отметкой

Место работ — точка погрузки, разгрузки, стоянки или транзита

Метка позиционирования — устройство для определения местоположения техники

КРВ (Карта Рабочего Времени) — визуализация использования времени техники

КИОВ — Коэффициент Использования Основного Времени

2. Справочники
2.1. Оборудование
Управление реестром самоходного горно-шахтного оборудования (ШАС, ПДМ) с моделями и техническими характеристиками

Просмотр списка оборудования
Открытие раздела Оборудование
Для просмотра списка всего оборудования выполните следующие действия:

Откройте главное меню навигации в левой части экрана

Нажмите на раздел «Справочники» для раскрытия подменю

Выберите пункт «Оборудование» из списка справочников

Дождитесь загрузки таблицы с данными об оборудовании

В таблице отобразятся все единицы техники со следующими колонками:

Гаражный номер — регистрационный номер техники

Тип — тип оборудования (ШАС или ПДМ)

Модель — название модели с техническими характеристиками

Идентификатор/номер — серийный заводской номер

Объем кузова/ковша, м³ — вместимость кузова самосвала или ковша погрузчика

Грузоподъемность, т. — максимальная масса груза в тоннах

Макс. скорость, км/ч — максимальная скорость движения

Объём бака, л. — емкость топливного бака в литрах

Дата ввода в эксплуатацию — дата начала использования техники

Дата вывода из эксплуатации — дата окончания эксплуатации (если установлена)

В верхней части таблицы отображается общее количество единиц оборудования. Таблица поддерживает бесконечную прокрутку — при достижении конца списка автоматически подгружаются следующие записи.

Фильтрация и поиск оборудования
Для поиска конкретного оборудования используйте встроенные фильтры:

Таблица автоматически отфильтрует результаты по мере ввода

Для сброса фильтра очистите поле поиска

Перетащите колонки мышью для изменения порядка отображения

Изменения сохраняются автоматически и применяются при следующем открытии

Создание новой модели оборудования
Добавление модели через таблицу оборудования
Модели оборудования содержат технические характеристики и используются при создании единиц техники. Для создания новой модели:

Откройте раздел «Справочники» → «Оборудование»

Нажмите зеленую кнопку «+ Добавить» в верхней панели инструментов

В появившейся форме заполните поля модели:

Название модели — введите наименование (например, «БелАЗ 75131», «CAT 777G»)

Максимальная скорость — укажите значение в км/ч (например, 60)

Объём бака — укажите емкость в литрах (например, 3800)

Грузоподъёмность — укажите в тоннах (например, 130)

Объём кузова/ковша — укажите в кубических метрах (например, 92)

Нажмите кнопку «Сохранить» внизу формы

Дождитесь сообщения «Объект добавлен в таблицу»

Внимание: Все числовые поля должны содержать положительные значения. Пустые поля будут сохранены как null.

После сохранения модель становится доступной для выбора при создании единиц техники.

https://screenshots/equipment/create_model_form.png
https://screenshots/equipment/model_saved_success.png

Редактирование модели оборудования
Для изменения характеристик существующей модели:

Найдите строку с нужной моделью в таблице оборудования

Кликните дважды по ячейке с названием модели для входа в режим редактирования

В выпадающем списке выберите ту же модель (это откроет форму редактирования)

Измените необходимые параметры:

Название модели — переименуйте при необходимости

Технические характеристики — обновите значения

Нажмите Enter или кликните вне поля для сохранения изменений

Изменения применятся ко всем единицам техники, использующим эту модель.

Альтернативный способ редактирования — изменить характеристики непосредственно в строке техники:

Дважды кликните по ячейке с объемом кузова, грузоподъемностью, скоростью или объемом бака

Введите новое значение

Нажмите Enter для подтверждения

Система автоматически обновит соответствующие поля связанной модели.

https://screenshots/equipment/edit_model_inline.png

Удаление модели оборудования
Для удаления неиспользуемой модели:

Кликните дважды по ячейке с названием модели

В выпадающем списке нажмите кнопку удаления (иконка корзины) рядом с названием модели

Подтвердите удаление в диалоговом окне:

Проверьте сообщение «Вы уверены, что хотите удалить модель: «{название}»?»

Нажмите кнопку «Удалить» для подтверждения или «Отмена» для отказа

Дождитесь завершения операции удаления

Внимание: Если модель используется хотя бы одной единицей техники, система может заблокировать удаление. Сначала отвязайте технику от модели или удалите технику.

https://screenshots/equipment/delete_model_confirm.png

Создание новой единицы оборудования
Добавление техники в реестр
Для регистрации новой единицы горно-шахтного оборудования в системе:

Откройте раздел «Справочники» → «Оборудование»

Нажмите зеленую кнопку «+ Добавить» в верхней панели инструментов

В появившейся форме заполните обязательные и дополнительные поля:

1. Тип техники — выберите из выпадающего списка:

ШАС (Шахтный Автосамосвал) — для самосвалов

ПДМ (Погрузочно-Доставочная Машина) — для погрузчиков

2. Модель — выберите существующую модель из списка или создайте новую:

Начните вводить название модели для поиска

Если модель отсутствует, введите новое название и нажмите «+ Создать»

При выборе модели автоматически заполнятся характеристики (скорость, объем бака и т.д.)

3. Серийный номер — введите уникальный заводской номер техники (до 100 символов)

4. Гаражный номер — укажите регистрационный номер (до 50 символов), например «СА-001»

5. Дата ввода в эксплуатацию — выберите дату начала использования (необязательно)

6. Дата вывода из эксплуатации — укажите плановую дату окончания эксплуатации (необязательно)

При необходимости отредактируйте технические характеристики модели прямо в форме:

Объем кузова/ковша, м³

Грузоподъемность, т.

Макс. скорость, км/ч

Объём бака, л.

Нажмите кнопку «Сохранить» внизу формы

Дождитесь сообщения «Объект добавлен в таблицу»

Проверьте появление новой строки в таблице оборудования

Примечания:

Название техники генерируется автоматически на основе типа и гаражного номера

Если при создании техники изменить характеристики модели, они автоматически обновятся для всей модели

Даты ввода/вывода можно оставить пустыми и заполнить позже

Серийный и гаражный номера должны быть уникальными в рамках предприятия

Найдите технику, которую нужно скопировать

Система создаст копию со следующими отличиями:

Новый уникальный ID

Пустой серийный номер (требуется ввести новый)

Те же модель, тип, гаражный номер и даты

Отредактируйте серийный номер и другие поля при необходимости

Сохраните изменения

Внимание: Поля created_at и updated_at не копируются — система установит новые значения автоматически.

Редактирование оборудования
Изменение параметров техники
Для редактирования существующей единицы оборудования:

Найдите нужную технику в таблице

Дважды кликните по любой редактируемой ячейке строки

Строка перейдет в режим редактирования — поля станут активными

Измените необходимые параметры:

Тип техники — выберите новый тип из выпадающего списка

Модель — выберите другую модель или создайте новую

Серийный номер — введите новый номер

Гаражный номер — измените регистрационный номер

Даты активности — скорректируйте период эксплуатации

Характеристики модели — обновите значения при необходимости

Для сохранения изменений:

Нажмите клавишу Enter

Или кликните вне редактируемой области

Или нажмите кнопку «Сохранить», если она отображается

Дождитесь сообщения «Изменения сохранены»

Особенности редактирования:

При изменении типа техники автоматически обновляется название

При смене модели подставляются её характеристики

Изменение характеристик в строке техники обновляет саму модель

Все изменения применяются немедленно после подтверждения

Если возникла ошибка, появится сообщение «Ошибка сохранения» с описанием проблемы

Изменение периода активности
Для установки или корректировки дат эксплуатации:

Дважды кликните по ячейке «Дата ввода в эксплуатацию»

В календаре выберите дату начала использования

Нажмите Enter для подтверждения

Дважды кликните по ячейке «Дата вывода из эксплуатации»

Выберите дату окончания (должна быть позже даты ввода)

Подтвердите выбор

Техника с установленной датой вывода считается неактивной и может исключаться из некоторых операций.

Массовое удаление техники
Для удаления нескольких единиц техники одновременно:

Примечание: Массовое удаление выполняется параллельно для ускорения процесса. В случае ошибки одна из операций может завершиться неудачно, но остальные будут выполнены.

Сортировка данных
Для упорядочивания списка техники:

Кликните по заголовку любой колонки для сортировки

Первый клик — сортировка по возрастанию (A-Z, 0-9)

По модели — для просмотра всей техники одной модели

По дате ввода — для выявления новой или старой техники

По грузоподъемности — для подбора техники под задачи

Бесконечная прокрутка
При большом количестве техники таблица использует бесконечную прокрутку:

Изначально загружается первая порция записей (обычно 50-100)

При прокрутке вниз до конца списка автоматически подгружаются следующие записи

Внизу таблицы отображается индикатор загрузки при подгрузке данных

Процесс продолжается пока не будут загружены все записи

Общее количество объектов отображается в верхней части таблицы

Это позволяет работать с большими реестрами техники без задержек при первоначальной загрузке.

https://screenshots/equipment/infinite_scroll.png

2.2. Статусы техники
Управление статусами для классификации рабочего времени и простоев техники

Просмотр списка статусов
Открытие раздела Статусы
Для просмотра всех доступных статусов выполните следующие действия:

Откройте главное меню навигации в левой части экрана

Нажмите на раздел «Справочники» для раскрытия подменю

Выберите пункт «Статусы» из списка справочников

Дождитесь загрузки таблицы со статусами

В таблице отобразятся все статусы со следующими колонками:

Наименование статуса — отображаемое название (например, «Работа», «Заправка», «Ремонт»)

Организационная категория — пользовательская категория для группировки (например, «Производство», «Обслуживание»)

Аналитическая категория — системная категория для расчетов (например, «Производственная (продуктивное время)», «Ремонт плановый»)

Как работает система позиционирования
Принцип работы
Система позиционирования работает следующим образом:

BLE-маяк (метка) транслирует свой сигнал в радиусе действия

Техника с бортовым оборудованием принимает сигнал маяка

Бортовое оборудование определяет уровень сигнала (RSSI)

Если сигнал достаточно сильный — техника находится внутри радиуса метки

Система фиксирует: какая техника, у какой метки, в какое время

https://screenshots/tags/positioning-principle.png

Определение местоположения
Когда техника входит в зону действия метки:

Система получает координаты метки (из привязанного места или из настроек)

Технике присваивается местоположение этой метки

На карте диспетчера техника отображается в точке метки

Фиксируется время входа в зону и время выхода из зоны

Использование меток в отчётах
Данные о позиционировании используются для:

Отслеживание маршрутов техники

Расчёт времени пребывания на местах работ

Анализ эффективности использования техники

Контроль выполнения маршрутных заданий

Формирование отчётов по работе участков

Работа с радиусом действия

Радиус действия метки отображается на карте:

На 2D и 3D картах зона действия показана прозрачным кругом/сферой

Цвет зоны: оранжевый (#FF6600) с прозрачностью 10%

Центр метки — яркий оранжевый шар диаметром 2.5 м

При выборе метки появляются 4 маркера по сторонам света для изменения радиуса

Рекомендации по радиусу
Как выбрать оптимальный радиус:

Слишком маленький радиус (<15 м) — техника может не определяться

Слишком большой радиус (>80 м) — теряется точность позиционирования

Для мест погрузки/разгрузки: 20-40 м (точная фиксация прибытия)

Для транзитных точек: 30-60 м (контроль прохождения маршрута)

Для стоянок: 40-80 м (общая зона парковки)

Теперь все отчёты по этому участку будут учитывать его как подрядный

Сценарий 3: Объединение нескольких горизонтов в один участок

Сценарий 4: Анализ производительности подрядчика
Задача: Нужно подготовить отчёт по работе подрядчика за месяц.

Откройте «Аналитика → Производительность по участкам»

В фильтре «Период» выберите нужный месяц

В фильтре «Тип организации» выберите «Подрядные»

В фильтре «Участок» выберите конкретного подрядчика

Отчёт будет содержать: объём работ, количество рейсов, время работы, простои

Часто задаваемые вопросы
Можно ли изменить название участка?
Да, название можно изменить в любой момент через форму редактирования. Новое название должно быть уникальным. Все связанные объекты (места, задания, отчёты) автоматически обновятся.

Что делать, если не получается удалить участок?
Система блокирует удаление участков, которые используются другими объектами. Проверьте:

Есть ли места работ, привязанные к этому участку — удалите или перепривяжите их

Есть ли активная техника на участке — снимите назначение

Есть ли незавершённые наряд-задания — завершите или отмените их

После устранения зависимостей повторите попытку удаления

Может ли участок не иметь привязанных горизонтов?
Технически да, но это не рекомендуется. Участок без горизонтов не сможет использоваться полноценно, так как места работ привязываются к горизонтам. Всегда указывайте хотя бы один горизонт при создании участка.

Как посмотреть все участки конкретного горизонта?
Используйте фильтр в таблице участков:

Откройте раздел «Участки»

В фильтре «Горизонты» выберите нужный горизонт

Таблица покажет только участки, включающие этот горизонт

Можно выбрать несколько горизонтов для одновременного просмотра

В чём разница между собственным и подрядным участком?
Основное отличие — в отчётности и аналитике:

Собственные участки — работы выполняются штатной техникой и персоналом предприятия

Подрядные участки — работы выполняются внешней организацией по договору

В отчётах данные группируются отдельно по типу организации

Можно сравнивать эффективность собственных и подрядных работ

Для подрядчиков формируются отдельные акты выполненных работ

Можно ли перенести места работ с одного участка на другой?
Да, это делается через редактирование места работ:

Откройте «Справочники → Места»

Найдите нужное место и откройте форму редактирования

В поле «Участок» выберите новый участок

Сохраните изменения

Место будет перепривязано к новому участку

Все исторические данные (рейсы, отчёты) сохранятся

14. Решение проблем
Типовые проблемы и способы их решения

Часто встречающиеся проблемы
Техника не отображается на карте
Проверьте наличие активного задания у техники

Убедитесь что техника активна в справочнике

Проверьте работу бортового оборудования

Проверьте подключение к серверу телеметрии

Обновите страницу браузера (F5)

Если проблема сохраняется — обратитесь к администратору

Рейсы не создаются автоматически
Проверьте наличие активного маршрутного задания

Убедитесь что техника начала движение

Проверьте работу машины состояний

Проверьте логи сервера диспетчеризации

Попробуйте создать рейс вручную

Если проблема системная — обратитесь к разработчикам

Неправильный расчет массы груза
Проверьте плотность вида груза в справочнике

Убедитесь что грузоподъемность модели указана верно

Проверьте объем кузова в параметрах модели

Пересчитайте массу по формуле: M = V × ρ

При необходимости скорректируйте параметры

Изменения применятся к новым рейсам

Метка позиционирования не работает
Проверьте уровень заряда батареи метки

Убедитесь что MAC-адрес указан правильно

Проверьте привязку метки к месту

Проверьте физическое наличие устройства

Замените батарею если заряд низкий

При поломке — замените метку на новую

Система работает медленно
Проверьте скорость интернет-соединения

Закройте лишние вкладки браузера

Очистите кэш браузера

Проверьте нагрузку на сервер

Уменьшите частоту обновления данных

При постоянной проблеме — обратитесь к администратору

Не удается сохранить изменения
Проверьте права доступа пользователя

Убедитесь что все обязательные поля заполнены

Проверьте формат введенных данных

Обновите страницу и повторите попытку

Проверьте подключение к серверу

Если ошибка повторяется — сообщите разработчикам
